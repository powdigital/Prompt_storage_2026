## How This Repo Is Organised

Each prompt lives in its own `.md` file inside the relevant category folder. Files follow a consistent naming convention:

```
category/prompt-name.md
```

Each prompt file includes:
- **What it does** — plain language description
- **Best model** — which model it works best on
- **The prompt** — the actual prompt, ready to copy
- **Example output** — optional, but useful for quick evaluation
- **Tags** — for cross-referencing

---

## Prompt Template

Use this template when adding a new prompt:

```markdown
# Prompt Name

## What it does
Brief description of the prompt's purpose and output.

## Best model
e.g. Claude Sonnet, GPT-4o, Gemini 1.5 Pro

## Prompt
\```
Your prompt goes here.
\```

## Example output
Optional — paste a sample output to show what good looks like.

## Tags
`tag1` `tag2` `tag3`
```
